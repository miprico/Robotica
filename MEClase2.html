teoría

¿Qué es Arduino?

01
🤖
La Plataforma Arduino
Arduino es una plataforma de electrónica abierta que combina hardware (una placa con microcontrolador) y software (el IDE Arduino) para crear proyectos interactivos. Es usada por millones de personas en todo el mundo: desde estudiantes hasta ingenieros.
02
📋
Tipos de Arduino
Existen muchos modelos: UNO (el más usado para aprender), Nano (pequeño), Mega (más pines), Leonardo, etc. En el Maker Lab trabajamos con el UNO porque tiene el tamaño ideal para aprender y tiene todos los pines necesarios para nuestros proyectos.
03
🔌
Partes del Arduino UNO
Las partes principales son: microcontrolador ATmega328P (el cerebro), pines digitales (0–13), pines analógicos (A0–A5), pines de alimentación (3.3V, 5V, GND), conector USB y conector de alimentación externa.
04
⚡
Pines Digitales
Los pines digitales (0–13) pueden ser entradas o salidas. En modo salida envían 5V (HIGH) o 0V (LOW). El pin 13 tiene un LED integrado en la placa. Siempre se deben configurar en setup() con pinMode().
05
💻
Programación por Bloques
Usamos Tinkercad o mBlock para programar con bloques visuales. Cada bloque representa una instrucción. Es la manera más visual de entender la lógica antes de escribir código. Los bloques se convierten automáticamente en código Arduino.
06
📝
Estructura de un Programa
Todo programa Arduino tiene dos funciones obligatorias: setup() que se ejecuta una sola vez al inicio (para configurar pines, inicializar componentes), y loop() que se ejecuta indefinidamente (toda la lógica del proyecto va aquí).

código base

Estructura básica de Arduino

💡
El código Arduino siempre tiene dos funciones: setup() para configurar y loop() para ejecutar continuamente. Sin estas dos funciones, el programa no funciona.
blink_basico.ino
// ─── Módulo 1 · Clase 2 · Blink LED ──────────────────────── // El programa más simple: hacer parpadear el LED del pin 13 void setup() { // setup() se ejecuta UNA VEZ al encender pinMode(13, OUTPUT); // pin 13 como SALIDA } void loop() { // loop() se repite PARA SIEMPRE digitalWrite(13, HIGH); // encender LED delay(1000); // esperar 1 segundo digitalWrite(13, LOW); // apagar LED delay(1000); // esperar 1 segundo }
⚠️
Importante: El delay se mide en milisegundos. 1000ms = 1 segundo. 500ms = medio segundo. Si pones delay(50) el LED parpadea tan rápido que parece siempre encendido.
experimentos

Los 3 experimentos de la clase

1
Blink LED

El experimento clásico de Arduino. Hacemos parpadear el LED interno de la placa (pin 13) usando digitalWrite() y delay().

  • Conectar el Arduino al computador con USB
  • Abrir el IDE de Arduino (o Tinkercad)
  • Escribir o cargar el código blink básico
  • Hacer clic en Subir (flecha →)
  • Observar el LED naranja parpadeando
  • Cambiar el valor del delay y subir de nuevo
2
Sirena de policía

Simulamos una sirena usando dos LEDs (rojo y azul) que parpadean en alternancia. Aprendemos a controlar varios pines y a crear patrones con delays diferentes.

  • Conectar LED rojo en pin 12 (con resistencia 220Ω)
  • Conectar LED azul en pin 11 (con resistencia 220Ω)
  • Configurar ambos pines como OUTPUT en setup()
  • Alternar los LEDs en loop() con delays de 200ms
  • Experimentar con diferentes velocidades
3
Semáforo

Recreamos un semáforo con 3 LEDs (rojo, amarillo, verde). Es la base del proyecto principal de la clase: el Semáforo doble. Practicamos la secuencia de estados y los tiempos reales.

  • Conectar LED rojo, amarillo y verde en pines 13, 12, 11
  • Configurar los 3 pines como OUTPUT
  • Encender verde 3s → amarillo 1s → rojo 3s
  • Apagar todos antes de cambiar de color
  • Ajustar los tiempos para que sea realista

Código: Semáforo completo

semaforo.ino
// ─── Módulo 1 · Clase 2 · Semáforo ───────────────────────── // Definir los pines de cada LED int rojo = 13; int amarillo = 12; int verde = 11; void setup() { pinMode(rojo, OUTPUT); pinMode(amarillo, OUTPUT); pinMode(verde, OUTPUT); } void loop() { // ── VERDE: circular ────────────────────── digitalWrite(verde, HIGH); delay(3000); // 3 segundos digitalWrite(verde, LOW); // ── AMARILLO: precaución ───────────────── digitalWrite(amarillo, HIGH); delay(1000); // 1 segundo digitalWrite(amarillo, LOW); // ── ROJO: alto ─────────────────────────── digitalWrite(rojo, HIGH); delay(3000); // 3 segundos digitalWrite(rojo, LOW); }

tips para la clase

Claves para el éxito

🔌
Resistencia siempre
Usar resistencia de 220Ω entre el pin y el LED. Sin ella el LED se daña en segundos.
📏
Polaridad del LED
Pata larga = ánodo (+) va al pin. Pata corta = cátodo (−) va a GND.
⏱️
Delay en milisegundos
1000 = 1 segundo. 500 = medio segundo. 100 = muy rápido. Practica con varios valores.
💾
Guardar siempre
Guardar el sketch antes de subir. Usar Ctrl+S. Nombrar el archivo con algo descriptivo.
🐛
El LED no enciende
Verificar: ¿el número de pin coincide en el código? ¿La resistencia está conectada? ¿El GND está conectado?
🧠
Programar por bloques
Si es tu primera vez, usa Tinkercad con bloques. Cada bloque muestra el código equivalente abajo.
