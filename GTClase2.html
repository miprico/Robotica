TEORÍA

¿Qué es Arduino?

Lee cada concepto y asegúrate de entenderlo antes de conectar

⏱ 15 min aprox.
🤖
La Plataforma
Arduino es una plataforma de electrónica abierta que combina una placa con microcontrolador y el software IDE. Permite crear proyectos interactivos que leen sensores, controlan LEDs, motores y más.
📋
Tipos de Arduino
Hay muchos modelos: UNO (el más usado para aprender), Nano (pequeño), Mega (muchos pines). En el Maker Lab usamos el UNO porque tiene el tamaño perfecto para empezar.
💻
Programación por bloques
Con Tinkercad o mBlock programamos arrastrando bloques visuales. Es la forma más fácil de entender la lógica. Cada bloque se convierte en código Arduino automáticamente.
📝
Estructura de un programa
Todo programa tiene dos funciones obligatorias: setup() que corre una sola vez al encender (para configurar pines), y loop() que se repite para siempre (donde va la lógica del proyecto).
⚡
Pines digitales
Los pines 0–13 son digitales: solo tienen dos estados, HIGH (5V = encendido) o LOW (0V = apagado). Se configuran con pinMode() y se controlan con digitalWrite().
⏱️
Función delay()
Pausa el programa por N milisegundos. delay(1000) = 1 segundo. delay(500) = medio segundo. Es la base del tiempo en Arduino.

🔵 Experimento 1 — Blink LED

El experimento clásico de Arduino. Vamos a hacer parpadear el LED integrado en el pin 13 sin necesidad de componentes externos.

blink_led.ino
// ── Experimento 1: Blink LED ───────────────────── void setup() { pinMode(13, OUTPUT); // pin 13 como SALIDA } void loop() { digitalWrite(13, HIGH); // encender delay(1000); // 1 segundo digitalWrite(13, LOW); // apagar delay(1000); // 1 segundo }

⚠️ delay = milisegundos: 1000 = 1 segundo. Con delay(50) el LED parpadea tan rápido que parece siempre encendido. ¡Pruébalo!
🟢 Experimento 2 — Sirena de policía

Dos LEDs (rojo y azul/otro color) alternándose rápidamente simulan la sirena de una patrulla. Materiales: 2 LEDs + 2 resistencias 220Ω + cables.

📌 Conexión: Pin 12 → Resistencia 220Ω → LED ROJO → GND | Pin 11 → Resistencia 220Ω → LED AZUL → GND
sirena.ino
// ── Experimento 2: Sirena de policía ───────────── int rojo = 12; int azul = 11; void setup() { pinMode(rojo, OUTPUT); pinMode(azul, OUTPUT); } void loop() { digitalWrite(rojo, HIGH); // rojo ON digitalWrite(azul, LOW); // azul OFF delay(200); digitalWrite(rojo, LOW); // rojo OFF digitalWrite(azul, HIGH); // azul ON delay(200); }
Progreso Experimento 20%
  • ✓
    Conecté el LED rojo en pin 12 con resistencia 220Ω
  • ✓
    Conecté el LED azul (o verde) en pin 11 con resistencia 220Ω
  • ✓
    Escribí el código y verifiqué sin errores
  • ✓
    Los LEDs se alternan — ¡parece una sirena!
  • ✓
    Cambié el delay a 100ms — ¿se ve más rápida?
🟠 Experimento 3 — Semáforo

El semáforo clásico con 3 LEDs y tiempos reales. Este experimento es la base directa del proyecto Semáforo doble. Materiales: LED rojo + amarillo + verde + 3 resistencias 220Ω.

// ── Experimento 3: Semáforo ────────────────────── int rojo = 13; int amarillo = 12; int verde = 11; void setup() { pinMode(rojo, OUTPUT); pinMode(amarillo, OUTPUT); pinMode(verde, OUTPUT); } void loop() { // VERDE — circular digitalWrite(verde, HIGH); delay(3000); digitalWrite(verde, LOW); // AMARILLO — precaución digitalWrite(amarillo, HIGH); delay(1000); digitalWrite(amarillo, LOW); // ROJO — alto digitalWrite(rojo, HIGH); delay(3000); digitalWrite(rojo, LOW); }

PROYECTO

Semáforo doble 🚦

Sincroniza dos semáforos para que funcionen como una intersección real

⏱ 20–30 min aprox.
RETO DEL PROYECTO

Cuando el Semáforo 1 está en verde, el Semáforo 2 debe estar en rojo, y viceversa. Los dos pasan por amarillo antes de cambiar. Así funciona una intersección real.
